Detailed Description

Capture-related error codes and explanations that you may receive through the delegate methods of the JRCaptureObjectDelegate, JRCaptureUserDelegate, and JRCaptureDelegate protocols.

enum JRCaptureApidError

Errors received from the JRCaptureObjectDelegate, JRCaptureUserDelegate, and JRCaptureDelegate protocols when they fail on the Capture server. These errors correspond to the errors listed on the Capture RESTful API Documentation Page

Enumerator:
JRCaptureApidErrorMissingArgument 

Error returned when a required argument was not supplied

JRCaptureApidErrorInvalidArgument 

Error returned when the argument was malformed, or its value was invalid for some other reason

JRCaptureApidErrorDuplicateArgument 

Error returned when two or more supplied arguments may not have been included in the same call; for example, both id and uuid in entity.update

JRCaptureApidErrorInvalidAuthMethod 

Error returned when the request used an http auth method other than Basic or OAuth

JRCaptureApidErrorUnknownApplication 

Error returned when the application id does not exist

JRCaptureApidErrorUnknownEntityType 

Error returned when the entity type does not exist

JRCaptureApidErrorUnknownAttribute 

Error returned when an attribute does not exist. This can occur when trying to create or update a record, or when modifying an attribute

JRCaptureApidErrorEntityTypeExists 

Error returned when the client attempts to create an entity type that already exists

JRCaptureApidErrorAttributeExists 

Error returned when the client attempts to create an attribute that already exists

JRCaptureApidErrorReservedAttribute 

Error returned when the client attempts to modify a reserved attribute; can occur if you try to delete, rename, or write to a reserved attribute

JRCaptureApidErrorRecordNotFound 

Error returned when the client refers to an entity or plural element that does not exist

JRCaptureApidErrorIdInNewRecord 

Error returned when the client attempts to specify a record id in a new entity or plural element

JRCaptureApidErrorTimestampMismatch 

Error returned when the created or lastUpdated value does not match the supplied argument

JRCaptureApidErrorInvalidDataFormat 

Error returned when a JSON value was not formatted correctly according to the attribute type in the schema

JRCaptureApidErrorInvalidJsonType 

Error returned when a value did not match the expected JSON type according to the schema

JRCaptureApidErrorInvalidDateTime
